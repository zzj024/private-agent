## 背景

v0.4 完成了前端重构 + 聊天历史注入。但整个闭环还缺两块：

1. **记忆注入**：已确认的记忆不会自动注入到聊天上下文，Agent 仍然"健忘"
2. **自动分级**：所有候选记忆都要人工审核，没有利用 LLM 自评的 confidence 做分流
3. **知识库管理**：只能手动输入目录路径导入，无浏览/搜索/编辑/删除功能

## 一、记忆注入

### 1.1 现状

```
用户发消息 → ChatService → LLM（只看到当前消息 + 历史对话）
                              ↑
                         记忆库完全没参与
```

### 1.2 目标

```
用户发消息 → ChatService → 拉取已确认记忆 → 拼入 system prompt → LLM
                              ↑
                      记忆库自动参与每次对话
```

### 1.3 实现位置

`app/chat_service.py` 的 `stream_events()` 方法，在加载历史消息之后、调用 agent 之前。

### 1.4 注入格式

在 `run_agent()` 的 `initial_state["messages"]` 最前面，插入一条 system-level 的 HumanMessage：

```python
def _build_memory_context(memories: list[dict]) -> str:
    """把记忆列表拼成注入文本"""
    if not memories:
        return ""
    
    lines = ["[系统记忆] 以下是关于用户的已知信息，请在回答时参考："]
    for m in memories:
        cat = m.get("category", "")
        key = m.get("key", "")
        value = m.get("value", "")
        lines.append(f"- [{cat}] {key}: {value}")
    return "\n".join(lines)
```

注入时机：加载历史 → 拉取记忆 → 把记忆文本插入 messages[0] → 再追加当前消息

### 1.5 影响范围

| 文件 | 改动 |
|---|---|
| `agent/graph.py` | `_load_conversation_messages()` 旁边新增 `_load_memory_context()` |
| `agent/graph.py` | `run_agent()` 接收并处理 memory_context |
| `agent/graph.py` | `run_agent_with_reflexion()` 透传 memory_context |
| `agent/reflexion.py` | `reflexion_loop()` / `generate_answer()` 透传 |
| `app/chat_service.py` | 调用 `_load_memory_context()` 并传入 |

### 1.6 测试验证

- [ ] 先保存一条记忆（通过审核接受），再聊天问相关问题，确认 LLM 引用了记忆
- [ ] 无记忆时正常聊天不受影响
- [ ] 大量记忆时不超出上下文窗口

## 二、自动分级

### 2.1 现状

```
LLM 提取候选 → 全部 pending → 用户逐条审核
```

### 2.2 目标

```
LLM 提取候选 → confidence ≥ 0.85 → 直接 accepted
             → 0.6 ≤ confidence < 0.85 → pending（人工审核）
             → confidence < 0.6 → 直接 rejected（不产生噪音）
```

### 2.3 实现位置

`memory/passive_extractor.py` 的 `extract_passive_memories()` 函数，在第 5 步"逐个写入候选"处。

### 2.4 改动逻辑

```python
# 现有代码
store.insert_memory_candidate(..., status="pending")

# 改为
confidence = float(c.get("confidence", 0.5))
if confidence >= 0.85:
    # 高可信：直接写入正式记忆
    store.save_memory(
        key=..., value=..., category=..., confidence=confidence,
        source="passive", evidence=..., ...,
    )
    # 同时记一条候选记录，方便追溯
    store.insert_memory_candidate(..., status="auto_accepted")
elif confidence >= 0.6:
    # 中可信：人工审核
    store.insert_memory_candidate(..., status="pending")
else:
    # 低可信：直接丢弃
    store.insert_memory_candidate(..., status="auto_rejected")
```

### 2.5 配置化

在 `config/settings.py` 中新增两个阈值配置：

```python
# 被动记忆自动分级阈值
passive_memory_auto_accept_threshold: float = 0.85   # ≥ 此值自动接受
passive_memory_auto_reject_threshold: float = 0.60   # < 此值自动拒绝
```

用户可在 `.env` 中覆盖：

```env
PASSIVE_MEMORY_AUTO_ACCEPT_THRESHOLD=0.85
PASSIVE_MEMORY_AUTO_REJECT_THRESHOLD=0.60
```

### 2.6 影响范围

| 文件 | 改动 |
|---|---|
| `config/settings.py` | 新增两个阈值字段 |
| `memory/passive_extractor.py` | `extract_passive_memories()` 加分级逻辑 |

### 2.7 测试验证

- [ ] confidence 0.9 → 直接出现在记忆库（accepted）
- [ ] confidence 0.7 → 出现在审核列表（pending）
- [ ] confidence 0.4 → 不出现在审核列表（auto_rejected）
- [ ] 阈值可通过 .env 配置调整

## 三、整体数据流（改完后）

```
用户发消息
  │
  ├─→ ChatService.stream_events()
  │     ├─ 加载历史消息（v0.4 已完成）
  │     ├─ 加载已确认记忆（v0.5 新增）──→ 注入 system prompt
  │     └─→ LLM 生成回复（带有完整上下文）
  │
  └─→ 消息落库 → 触发被动提取
        │
        ├─ LLM 分析对话提取候选
        │
        └─ 自动分级（v0.5 新增）
              ├─ high → 自动写入正式记忆
              ├─ mid  → 待审核（前端 Review 列表）
              └─ low  → 自动丢弃
```
